Integrating Your Changelog with Slack and Discord

Your users are already in Slack and Discord. Meet them there with automated changelog notifications that keep your community informed and engaged.

Your changelog lives on your website, but your users live in Slack and Discord. Every time you publish an update, you're missing an opportunity to reach your community where they already hang out.

The solution? Webhook integrations that automatically post changelog updates to your team's communication channels. In this guide, we'll show you exactly how to set up Slack and Discord integrations—no complex infrastructure required.

Why Integrate Your Changelog with Chat?

Before diving into the technical setup, let's establish why this integration matters:

  • Reach users where they are. 80% of SaaS teams use Slack or Discord daily. Your changelog notifications become part of their existing workflow.
  • Increase visibility. Email open rates average 20-30%. Chat notifications get 60-80% view rates within the first hour.
  • Drive immediate engagement. Users can click through, try new features, and give feedback—all without leaving their workspace.
  • Build community. Public changelog channels create a space for users to discuss updates, share use cases, and feel part of your product's evolution.
  • Reduce support tickets. When users know what's changed, they encounter fewer surprises and submit fewer "is this a bug?" questions.
"The moment we connected our changelog to Slack, engagement with our updates tripled. Users were actually reading what we shipped."
— Founder at a B2B SaaS startup

Understanding Webhooks

Both Slack and Discord use webhooks to receive external messages. Think of a webhook as a special URL that accepts formatted messages and posts them to a specific channel.

Here's how the flow works:

  1. You create a webhook URL in Slack or Discord
  2. Your changelog system sends a POST request to that URL when a new entry is published
  3. The platform receives the message and posts it to the designated channel

No polling, no complex APIs—just a simple HTTP request that triggers a notification.

Slack Integration

💬 Slack Webhook Setup

Slack's Incoming Webhooks feature lets you post messages from external sources into Slack channels.

1

Create a Slack App

  1. Go to api.slack.com/apps
  2. Click Create New App
  3. Choose From scratch
  4. Name your app (e.g., "Changelog Bot") and select your workspace
  5. Click Create App
2

Enable Incoming Webhooks

  1. In your app's settings, navigate to Incoming Webhooks
  2. Toggle Activate Incoming Webhooks to On
  3. Scroll down and click Add New Webhook to Workspace
  4. Select the channel where changelog updates should be posted
  5. Click Allow
3

Copy Your Webhook URL

After authorization, you'll see a webhook URL that looks like this:

YOUR_SLACK_WEBHOOK_URL

Keep this URL secure. Anyone with this URL can post messages to your channel.

Formatting Slack Messages

Slack supports rich formatting via Block Kit, a JSON-based UI framework. Here's a well-formatted changelog notification:

Slack Payload Example
{ "blocks": [ { "type": "header", "text": { "type": "plain_text", "text": "🚀 New Feature: One-Click Exports" } }, { "type": "section", "text": { "type": "mrkdwn", "text": "*What's new:* Export your reports in PDF, CSV, or Excel format with a single click. No more copy-pasting into spreadsheets." } }, { "type": "section", "fields": [ { "type": "mrkdwn", "text": "*Category*\n🆕 New Feature" }, { "type": "mrkdwn", "text": "*Date*\nApril 8, 2026" } ] }, { "type": "actions", "elements": [ { "type": "button", "text": { "type": "plain_text", "text": "Read Full Update" }, "url": "https://changelog.page/updates/one-click-exports", "style": "primary" }, { "type": "button", "text": { "type": "plain_text", "text": "View Documentation" }, "url": "https://docs.example.com/exports" } ] } ] }

Sending the Webhook Request

You can send this payload using any HTTP client. Here's a cURL example:

cURL Command
curl -X POST -H 'Content-type: application/json' \ --data '@slack-payload.json' \ YOUR_SLACK_WEBHOOK_URL

And here's a Node.js example:

Node.js
const axios = require('axios'); async function notifySlack(webhookUrl, changelogEntry) { const payload = { blocks: [ { type: 'header', text: { type: 'plain_text', text: `🚀 ${changelogEntry.title}` } }, { type: 'section', text: { type: 'mrkdwn', text: changelogEntry.summary } }, { type: 'actions', elements: [ { type: 'button', text: { type: 'plain_text', text: 'Read More' }, url: changelogEntry.url, style: 'primary' } ] } ] }; await axios.post(webhookUrl, payload); }

Discord Integration

🎮 Discord Webhook Setup

Discord's webhook system is simpler than Slack's but still supports rich embeds with images, fields, and color coding.

1

Create a Discord Webhook

  1. Open your Discord server settings
  2. Navigate to Integrations → Webhooks
  3. Click New Webhook
  4. Name your webhook (e.g., "Changelog")
  5. Select the channel where updates should be posted
  6. Click Copy Webhook URL
2

Customize the Avatar (Optional)

Upload a custom avatar for your webhook to make it recognizable. Your product logo works great here.

Formatting Discord Messages

Discord supports rich embeds with colors, fields, images, and timestamps. Here's an example:

Discord Payload Example
{ "content": "📢 New changelog entry published!", "embeds": [ { "title": "🚀 One-Click Exports", "description": "Export your reports in PDF, CSV, or Excel format with a single click. No more copy-pasting into spreadsheets.", "url": "https://changelog.page/updates/one-click-exports", "color": 65280, "fields": [ { "name": "Category", "value": "🆕 New Feature", "inline": true }, { "name": "Date", "value": "April 8, 2026", "inline": true } ], "image": { "url": "https://changelog.page/images/export-demo.png" }, "footer": { "text": "Changelog.page" }, "timestamp": "2026-04-08T10:00:00.000Z" } ] }
Color Codes for Discord Embeds
New Feature: 65280 (Green) Improvement: 3447003 (Blue) Bug Fix: 16776960 (Yellow) Security: 15158332 (Red) Deprecation: 10070709 (Gray)

Sending the Discord Webhook Request

Node.js
const axios = require('axios'); async function notifyDiscord(webhookUrl, changelogEntry) { const colorMap = { 'new': 65280, 'improved': 3447003, 'fixed': 16776960, 'security': 15158332 }; const payload = { content: '📢 New changelog entry published!', embeds: [{ title: changelogEntry.title, description: changelogEntry.summary, url: changelogEntry.url, color: colorMap[changelogEntry.category] || 3447003, fields: [ { name: 'Category', value: changelogEntry.category, inline: true }, { name: 'Date', value: changelogEntry.date, inline: true } ], timestamp: new Date().toISOString() }] }; await axios.post(webhookUrl, payload); }

Automation: Triggering Webhooks on Publish

Manually sending webhooks every time you publish a changelog entry doesn't scale. Here's how to automate it:

Option 1: GitHub Actions

If your changelog is stored in a GitHub repository, use GitHub Actions to trigger webhooks when new entries are merged:

.github/workflows/notify.yml
name: Notify on Changelog Update on: push: paths: - 'changelog/*.md' branches: [main] jobs: notify: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Get latest changelog entry id: latest run: | LATEST=$(ls -t changelog/*.md | head -1) echo "file=$LATEST" >> $GITHUB_OUTPUT - name: Send Slack notification env: SLACK_WEBHOOK: ${{ secrets.SLACK_WEBHOOK_URL }} run: | curl -X POST -H 'Content-type: application/json' \ --data '{"text":"New changelog entry published!"}' \ $SLACK_WEBHOOK

Option 2: Build Script Integration

Add webhook calls to your changelog build script:

build.js
const fs = require('fs'); const axios = require('axios'); async function buildAndNotify() { // Build your changelog await buildChangelog(); // Get the latest entry const entries = await getChangelogEntries(); const latest = entries[0]; // Send notifications if (process.env.SLACK_WEBHOOK_URL) { await notifySlack(process.env.SLACK_WEBHOOK_URL, latest); } if (process.env.DISCORD_WEBHOOK_URL) { await notifyDiscord(process.env.DISCORD_WEBHOOK_URL, latest); } } buildAndNotify().catch(console.error);

Option 3: RSS Feed Polling

If your changelog has an RSS feed, use a service like Zapier or Make to watch for new entries and trigger webhooks automatically:

  1. Set up a Zapier "RSS by Zapier" trigger pointing to your changelog RSS feed
  2. Add a "Webhooks by Zapier" action to POST to your Slack/Discord webhook URL
  3. Map RSS fields (title, description, link) to your webhook payload

Best Practices for Chat Notifications

Getting the integration working is just the start. Here's how to make your notifications truly effective:

Keep It Concise

Chat messages should be scannable. Include the title, a one-sentence summary, and a link to read more. Don't paste the full changelog entry into the channel.

Use Consistent Formatting

Create a template and stick to it. Users should instantly recognize a changelog notification and know what to expect.

Include Direct Links

Every notification should link to the full changelog entry and, when relevant, to documentation or the feature itself. Reduce friction for users who want to explore.

Respect Channel Context

Channel TypeFrequencyTone
#changelog (dedicated)Every updateInformative, detailed
#generalMajor features onlyBrief, celebratory
#product-updatesWeekly digestSummary, links

Handle Errors Gracefully

Webhooks can fail. Implement retry logic and log failures so you know when notifications aren't going through:

Retry Logic
async function sendWithRetry(webhookUrl, payload, retries = 3) { for (let i = 0; i < retries; i++) { try { await axios.post(webhookUrl, payload); return; } catch (error) { if (i === retries - 1) throw error; await new Promise(r => setTimeout(r, 1000 * (i + 1))); } } }

Security Considerations

  • Store webhook URLs in environment variables or secrets managers—never commit them to your repository
  • Rotate webhook URLs periodically if you suspect they've been exposed
  • Validate payloads before sending to prevent injection attacks
  • Use HTTPS for all webhook URLs

Real-World Examples

Supabase: Community-First Discord

Supabase posts every changelog entry to their Discord server, where their active community discusses new features in real-time. This approach has turned their changelog into a community engagement tool, with users often replying with feature requests and bug reports within minutes of a post.

Vercel: Multi-Channel Slack Strategy

Vercel uses a tiered approach: major releases get posted to their public #announcements channel, while all updates flow to a dedicated #changelog channel. This lets users choose their level of engagement without overwhelming general channels.

Linear: Internal + External

Linear posts changelog updates to both an internal Slack channel (for team awareness) and a public Discord server (for community transparency). The internal posts include technical details, while the public versions focus on user-facing benefits.

Key Takeaways

  • Meet users where they are. Slack and Discord notifications reach users in their existing workflow, dramatically increasing visibility.
  • Webhooks are simple but powerful. A single HTTP POST is all it takes to bridge your changelog and chat platforms.
  • Format for the medium. Chat notifications should be concise, scannable, and include clear calls-to-action.
  • Automate the flow. Use GitHub Actions, build scripts, or RSS polling to trigger notifications without manual work.
  • Secure your webhooks. Treat webhook URLs like passwords—store them securely and rotate them periodically.
  • Match frequency to context. Dedicated changelog channels can handle every update; general channels should only see major announcements.
  • Use color coding. Consistent colors for categories (new, fixed, security) make notifications instantly scannable.
