## How to Use This Template

1. Copy this file to your changelog content directory
2. Replace the placeholder content with your actual updates
3. Update the YAML frontmatter (date, title, categories)
4. Remove any sections you don't need
5. Build and deploy

## Tips for This Template

✅ **DO:**
- Lead with the user benefit, not the technical details
- Use emojis to make scanning easier
- Include specific metrics when possible
- Link to documentation for complex features
- Use tables for bug fixes (easy to scan)

❌ **DON'T:**
- Use internal codenames without explanation
- List every minor bug fix (group them or skip)
- Write more than 500 words per major feature
- Forget to proofread before publishing
