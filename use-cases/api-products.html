## New Webhook Events

We've added two new webhook events to help you better handle payment failures and refunds:

### payment.failed

Triggered when a charge fails. Includes:
- `failure_code` - Machine-readable failure reason
- `failure_message` - Human-readable description
- `next_payment_attempt` - Timestamp for retry

```json
{
  "type": "payment.failed",
  "data": {
    "charge_id": "ch_123456",
    "failure_code": "card_expired",
    "next_payment_attempt": "2024-01-16T10:00:00Z"
  }
}
```

### payment.refunded

Triggered when a refund is processed. [View docs →](/docs/webhooks)
