# Product Hunt Launch Assets

Complete launch package for Changelog.page Product Hunt debut.

---

## 📁 Contents

| File | Description |
|------|-------------|
| `launch-copy.md` | Headline, description, maker story, and links |
| `response-templates.md` | Pre-written responses for common comment types |
| `launch-day-plan.md` | Hour-by-hour playbook for launch day |
| `generate-screenshots.js` | Node.js script to create 5 gallery images |
| `screenshots/` | Generated 1200x800 images (run script first) |

---

## 🚀 Quick Start

### 1. Review and Customize Copy

Read `launch-copy.md` and personalize:
- [ ] Update any [placeholder] links
- [ ] Adjust the maker story to your voice
- [ ] Confirm pricing is accurate

### 2. Generate Screenshots

```bash
cd product-hunt-launch
npm install canvas
node generate-screenshots.js
```

This creates 5 images in `screenshots/`:
1. `01-hero-dashboard.png` — Main dashboard
2. `02-themes-comparison.png` — 3 theme previews
3. `03-markdown-editor.png` — Split-pane editor
4. `04-custom-domain.png` — Domain settings
5. `05-mobile-responsive.png` — Multi-device preview

### 3. Read the Launch Day Plan

Study `launch-day-plan.md` for:
- Pre-launch checklist
- Supporter outreach templates
- Hour-by-hour schedule
- Emergency protocols

### 4. Schedule Your Launch

1. Go to [producthunt.com](https://producthunt.com)
2. Click "Submit a product" (top right)
3. Fill in details from `launch-copy.md`
4. Upload screenshots from `screenshots/`
5. Schedule for optimal day (Tuesday-Thursday)

## 💬 Response Strategy

During launch day, keep `response-templates.md` open. Key tips:

1. **Respond within 5 minutes** during peak hours
2. **Use their name** — personalize every response
3. **Ask questions** — keep conversations going
4. **Be grateful** — even for criticism
5. **Tag people** — use @username to notify
