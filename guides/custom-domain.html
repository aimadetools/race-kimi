Custom Domain Setup

Host your changelog on your own domain for a professional, branded experience. This guide covers DNS configuration, SSL certificates, and deployment on popular platforms.

Prerequisites: You'll need a registered domain name. If you don't have one yet, we recommend Porkbun, Namecheap, or Cloudflare Registrar.

🌐 How It Works

When you set up a custom domain for your changelog, here's what happens behind the scenes:

  1. You configure DNS — Create records that point your domain to your hosting platform
  2. You add the domain — Tell your hosting platform about your custom domain
  3. SSL is provisioned — The platform automatically sets up HTTPS certificates
  4. Traffic is routed — Visitors to your domain see your changelog

The entire process usually takes 5-15 minutes, though DNS propagation can take up to 24 hours in some cases.

⚙️ Configure DNS Records

You'll need to add DNS records at your domain registrar. The exact records depend on whether you want to use a root domain or a subdomain.

Option A: Subdomain (Recommended)

Use a subdomain like changelog.example.com or updates.example.com. This is easier to set up and doesn't affect your main website.

Type Name Value TTL
CNAME changelog See platform-specific value below Auto

Vercel CNAME value: cname.vercel-dns.com

Netlify CNAME value: Your site's Netlify subdomain (e.g., yoursite.netlify.app)

GitHub Pages CNAME value: username.github.io

Cloudflare Pages CNAME value: Your Cloudflare Pages subdomain

Option B: Root Domain (APEX)

Use your root domain like example.com. This requires special DNS records.

➕ Add Domain to Your Platform

Adding a Domain on Vercel

  1. Go to your Vercel Dashboard
  2. Select your changelog project
  3. Click Settings → Domains
  4. Enter your domain (e.g., changelog.example.com)
  5. Click Add

Vercel will automatically detect your DNS configuration and provision an SSL certificate. This usually takes 1-2 minutes.

Adding a Domain on Netlify

  1. Go to your Netlify Dashboard
  2. Select your site
  3. Go to Site settings → Domain management
  4. Click Add custom domain
  5. Enter your domain and click Verify
  6. Click Add domain

Netlify will provide you with the exact CNAME value to use in your DNS settings.

Adding a Domain on GitHub Pages

  1. Create a file named CNAME in your repository root
  2. Add your domain as the only content: changelog.example.com
  3. Commit and push the file
  4. Go to your repository → Settings → Pages
  5. Under "Custom domain", enter your domain
  6. Click Save

GitHub will check your DNS configuration. Once verified, HTTPS will be automatically enabled.

Adding a Domain on Cloudflare Pages

  1. Go to your Cloudflare Dashboard
  2. Select your Pages project
  3. Go to Custom domains
  4. Click Set up a custom domain
  5. Enter your domain and click Continue
  6. Add the CNAME record shown in your DNS settings

Cloudflare will verify the DNS record and issue an SSL certificate automatically.

✅ Verify & Test

After setting up your DNS and adding your domain to your platform, here's how to verify everything is working:

Checklist

  • Visit your custom domain in a browser
  • Confirm the page loads correctly
  • Check that HTTPS is working (look for the lock icon)
  • Verify your changelog content is displayed
  • Test navigation links

DNS Propagation Check

Use these tools to check if your DNS has propagated:

  • whatsmydns.net — Check DNS propagation globally
  • dnschecker.org — Verify DNS records
  • dig changelog.example.com — Command line check
✓ Success! If your changelog loads at your custom domain with HTTPS, you're all set!

🔧 Troubleshooting

"Domain not found" or 404 Error

  • Wait for DNS propagation. DNS changes can take up to 24 hours, though usually complete within 1 hour.
  • Clear your browser cache. Browsers cache DNS results aggressively.
  • Check your DNS records. Ensure the CNAME/A record points to the correct value for your platform.

SSL Certificate Errors

  • Wait for certificate provisioning. SSL certificates can take 5-30 minutes to issue after DNS is configured.
  • Verify domain ownership. Some platforms require you to verify you own the domain before issuing certificates.
  • Check for mixed content. Ensure all resources (images, CSS) use HTTPS URLs.

Redirect Loops

  • Check for conflicting redirects. Ensure you're not redirecting both HTTP→HTTPS and HTTPS→HTTP.
  • Verify CNAME chain. Some registrars have issues with CNAME flattening. Try using the platform's DNS service.
