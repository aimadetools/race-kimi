Setup Guide

1

Get Your Linear API Token

First, you need a Linear API token to authenticate with your workspace.

  1. Go to Linear → Settings → API
  2. Click "Create new token"
  3. Name it "Changelog Sync"
  4. Copy the token (starts with lin_api_)
Environment Variable 
# Add to your .env file or GitHub Secrets
LINEAR_API_TOKEN=lin_api_your_token_here
2

Find Your Team or Project ID

Use the CLI to explore your Linear workspace and find the team or project you want to sync.

Explore Workspace 
# Navigate to the generator directory
cd generator/src

# Run the info command
node linear.js --token $LINEAR_API_TOKEN --info

This will show all your teams, their states, and available projects.

3

Configure the Sync

Create a configuration file or use environment variables to set up the sync parameters.

linear.config.js 
module.exports = {
  // Required: Your Linear API token
  linearToken: process.env.LINEAR_API_TOKEN,
  
  // Optional: Filter by team ID (from --info output)
  teamId: 'team-uuid-here',
  
  // Optional: Filter by project ID
  projectId: 'project-uuid-here',
  
  // Optional: Only sync specific states
  stateNames: ['Done', 'Completed', 'Shipped'],
  
  // Required: Output directory for markdown files
  outputDir: './entries',
  
  // Optional: Maximum number of issues
  limit: 100,
  
  // Optional: Custom category mapping
  customCategoryMap: {
    'Done': 'feature',
    'Bug Fix': 'fix',
    'Improvement': 'improvement'
  }
};
4

Run the Sync

Sync issues from Linear to your changelog entries directory.

Manual Sync 
# Sync with configuration file
node linear.js --token $LINEAR_API_TOKEN \
  --team team-uuid \
  --states "Done,Completed" \
  --output ./entries

This will create markdown files in your entries directory, ready to be built into your changelog.

GitHub Actions Workflow

Set up automated sync to run daily or trigger when issues are completed.

.github/workflows/sync-linear.yml 
name: Sync Linear to Changelog

on:
  # Run daily at 9 AM
  schedule:
    - cron: '0 9 * * *'
  
  # Or run manually
  workflow_dispatch:

jobs:
  sync:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Sync Linear issues
        env:
          LINEAR_API_TOKEN: ${{ secrets.LINEAR_API_TOKEN }}
        run: |
          node generator/src/linear.js \
            --token "$LINEAR_API_TOKEN" \
            --team "${{ vars.LINEAR_TEAM_ID }}" \
            --states "Done,Completed" \
            --output ./entries \
            --limit 50
      
      - name: Commit changes
        run: |
          git config user.name "Linear Sync"
          git config user.email "linear@changelog.page"
          git add entries/
          git diff --quiet && git diff --staged --quiet || \
            git commit -m "Sync Linear issues: $(date +%Y-%m-%d)"
          git push

Data Mapping Reference

Here's how Linear data maps to changelog fields:

Linear Field Changelog Field Notes
title title Issue title becomes entry title
state.type category completed → feature, canceled → fix, etc.
labels tags Label names become tags
completedAt date Completion date or updatedAt fallback
assignee.name author Who completed the issue
description content Issue description (markdown)
team.name team Linear team name
project.name project Project the issue belongs to
url Reference link Link back to original Linear issue

Default State to Category Mapping

Linear State Type Changelog Category
completed feature
canceled fix
started improvement
unstarted improvement
triage improvement
backlog improvement

CLI Reference

Commands

Available Options 
# Show workspace info (teams, projects, states)
node linear.js --token <token> --info

# Sync with all options
node linear.js \
  --token <token> \
  --team <team-id> \
  --project <project-id> \
  --states "Done,Completed" \
  --output ./entries \
  --limit 100

# Show help
node linear.js --help

Options

Option Required Description
--token, -t Yes Linear API token (or LINEAR_API_TOKEN env var)
--team No Filter by team ID
--project No Filter by project ID
--states No Comma-separated state names to include
--output, -o No Output directory (default: ./entries)
--limit No Maximum issues to sync (default: 100)
--info No Show workspace info and exit
--help, -h No Show help message
