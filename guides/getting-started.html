Getting Started

Create your first changelog in 5 minutes. This guide will take you from installation to a deployed changelog page.

What You'll Build

By the end of this guide, you'll have:

  • A working changelog generator on your computer
  • Your first changelog entry written in Markdown
  • A beautiful HTML changelog page
  • Automatic RSS and JSON feeds
  • The knowledge to deploy to Vercel or GitHub Pages

Prerequisites

Before you begin, make sure you have:

  • Node.js 18+ — Download here
  • Git — For version control and deployment
  • A code editor — VS Code recommended
  • A GitHub account — For hosting and deployment
1
Clone the Repository

Start by cloning the changelog generator to your local machine:

git clone https://github.com/aimadetools/race-kimi.git
cd race-kimi/generator

This gives you the complete generator with three built-in themes and example content.

2
Install Dependencies

Install the required Node.js packages:

npm install

This installs the Markdown parser, YAML frontmatter processor, and build tools.

3
Create Your First Entry

Create a new Markdown file in the content/ directory. The filename format is YYYY-MM-DD-title.md:

touch content/2026-04-08-hello-world.md

Open the file in your editor and add this content:

---
title: "Hello World — Our First Changelog Entry"
date: 2026-04-08
category: Feature
tags: [launch, announcement]
author: Your Name
---

We're excited to launch our new changelog! This is where we'll share all our product updates, new features, and improvements.

## What's New

- **Beautiful Design** — Clean, readable layout optimized for your users
- **RSS Feed** — Subscribe to stay updated on every release
- **Markdown Powered** — Write in the format developers love

Stay tuned for more updates!
4
Build Your Changelog

Run the build command to generate your changelog:

npm run build

This creates a dist/ folder with your generated changelog:

📁 dist/
├── 📄 index.html (main changelog page)
├── 📄 hello-world.html (individual entry)
├── 📄 rss.xml (RSS feed)
├── 📄 feed.json (JSON feed)
└── 📁 css/
5
Preview Locally

Start a local server to preview your changelog:

npm run serve

Open http://localhost:3000 in your browser to see your changelog.

💡 Pro Tip

The server automatically reloads when you make changes. Keep it running while you edit your content!

Choosing a Theme

The generator includes three beautiful themes. Switch between them using the THEME environment variable:

Minimal (Default)

npm run build

Clean, typography-focused design. Best for technical audiences.

Cards

THEME=cards npm run build

Visual, card-based layout with hover effects. Great for product-focused updates.

Timeline

THEME=timeline npm run build

Vertical timeline layout. Perfect for showing product evolution.

Deploying Your Changelog

Once you're happy with your changelog, it's time to deploy. Here are your options:

Option 1: Vercel (Recommended)

Deploy to Vercel for free with automatic updates on every push:

  1. Push your code to GitHub
  2. Import your repository in Vercel
  3. Set the root directory to generator
  4. Deploy!

Option 2: GitHub Pages

Use the included GitHub Action for automatic deployment:

  1. Enable GitHub Pages in your repository settings
  2. Select "GitHub Actions" as the source
  3. The included workflow will deploy on every push to content/
🎉 Success!
