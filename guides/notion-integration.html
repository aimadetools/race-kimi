How It Works

The Notion integration fetches pages from your Notion database, converts the content to markdown format, and generates changelog entries that work with all Changelog.page themes.

Note: This feature requires a Team tier subscription ($49/mo) which includes API access and premium integrations.

Setup Guide

1

Create a Notion Integration

First, you need to create a Notion integration to get an API token:

  1. Go to notion.so/my-integrations
  2. Click "New integration"
  3. Name it "Changelog.page Sync"
  4. Select the workspace where your database lives
  5. Copy the "Internal Integration Token"
2

Share Your Database

Your integration needs access to your database:

  1. Open your Notion database
  2. Click the "..." menu in the top right
  3. Click "Add connections"
  4. Search for "Changelog.page Sync" and select it
3

Get Your Database ID

Copy your database ID from the Notion URL:

# URL format:
https://www.notion.so/workspace/XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX?v=...

# The database ID is:
XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX
4

Set Up Your Database Schema

Create these properties in your Notion database:

Property Name Type Required Description
Name Title ✓ Entry title
Category Select ✓ Feature, Fix, Improvement, etc.
Date Date ✓ Publication date
Tags Multi-select — Entry tags
Author Rich text — Entry author
5

Configure Environment Variables

Add these to your .env file or GitHub Actions secrets:

NOTION_TOKEN=secret_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NOTION_DATABASE_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
6

Update Your Build Script

Add Notion sync to your build process in package.json:

{
  "scripts": {
    "sync:notion": "node generator/src/notion-sync.js",
    "build": "npm run sync:notion && node generator/src/build.js"
  }
}

GitHub Actions Integration

Automate the sync process with GitHub Actions:

name: Sync from Notion

on:
  schedule:
    - cron: '0 */6 * * *'  # Every 6 hours
  workflow_dispatch:       # Manual trigger

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: npm install
      
      - name: Sync from Notion
        env:
          NOTION_TOKEN: ${{ secrets.NOTION_TOKEN }}
          NOTION_DATABASE_ID: ${{ secrets.NOTION_DATABASE_ID }}
        run: npm run sync:notion
      
      - name: Commit changes
        run: |
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add content/
          git diff --staged --quiet || git commit -m "Sync changelog entries from Notion"
          git push

Supported Notion Blocks

The integration converts these Notion blocks to markdown:

  • Paragraphs → Plain text
  • Headings 1-3 → Markdown headings (# ## ###)
  • Bulleted lists → Markdown bullet points
  • Numbered lists → Markdown numbered lists
  • To-do lists → Markdown checkboxes
  • Code blocks → Fenced code blocks with syntax highlighting
  • Quotes → Markdown blockquotes
  • Callouts → Blockquote with emoji prefix
  • Dividers → Horizontal rules

Note: Images, tables, and embeds are currently not supported. Use these blocks in your Notion pages, but they won't appear in the changelog output.

Custom Property Mapping

If your Notion database uses different property names, you can customize the mapping:

// generator/src/notion-sync.js
const { syncFromNotion } = require('./src/notion');

const config = {
  notionToken: process.env.NOTION_TOKEN,
  notionDatabaseId: process.env.NOTION_DATABASE_ID,
  outputDir: './generator/content',
  propertyMapping: {
    title: 'Title',           // Your title property name
    category: 'Type',         // Your category property name
    date: 'Published',        // Your date property name
    tags: 'Labels',           // Your tags property name
    author: 'Created By'      // Your author property name
  }
};

syncFromNotion(config)
  .then(() => console.log('✅ Sync complete'))
  .catch(err => {
    console.error('❌ Sync failed:', err);
    process.exit(1);
  });

Filtering Entries

Only sync entries that match specific criteria:

const config = {
  // ... other config
  filter: {
    property: 'Status',
    select: {
      equals: 'Published'
    }
  }
};

Migration from Notion

If you're currently using Notion for your changelog and want to migrate fully to Changelog.page:

  1. Set up the integration to sync all existing entries
  2. Run the sync once to generate all markdown files
  3. Review and adjust the generated files as needed
  4. Disable the sync and maintain entries directly in markdown
  5. Or keep using Notion and sync periodically

Read our complete guide on creating a changelog in Notion for more details on setting up your database structure.
